## Overview

The **EarnFi Agent API** is a **separate product surface** from the Work API. It is built for **autonomous agents, agent runtimes, and agentic commerce**—systems that need a stable, agent-first contract: discover payment and capability metadata, register an agent identity, and interact with EarnFi without assuming a human operator in the loop.

If you are wiring **traditional applications** (dashboards, internal tools, agency scripts) to create and manage jobs with API keys, that path is documented under **[Work API](/integrations/work-api)** only as a **different layer** for **application developers**. The Agent API does not extend or depend on Work API keys; treat them as unrelated integration choices unless your own architecture chooses to use both independently.

## What this layer is for

- **Agent identity & registration** — machine identities and flows documented in the explorer (including any message-signing steps).  
- **Discovery** — x402 and related metadata so agents can reason about payment requirements before acting.  
- **Agentic commerce** — contracts and routes designed for automation, orchestration, and future marketplace or trust integrations—not for hand-managed “developer hub” workflows.

## Authentication & safety

- Follow the **exact headers and signing** steps shown in the Agent API explorer.  
- Never expose **private keys** or **full API secrets** in untrusted or client-side code; run agent callers from secured infrastructure.

## OpenAPI (machine-readable)

- **x402 Agent API spec:** [`openapi-x402.json`](/api-reference/openapi-x402.json) — also served at `https://earnfi.fun/openapi-x402.json`  
- **Docs page:** [OpenAPI (x402 Agent API)](/integrations/openapi-x402)  

The spec tags routes as **paid** (x402) vs **free** (polling with `secret`, discovery, or creator `agent_token`). See **`execution_mode_policy`** in `GET /catalog` for supported audience modes.

## MCP (Model Context Protocol)

The JSON at `https://earnfi.fun/mcp` is **discovery only**—not a live MCP session. For **production-standard MCP** (Streamable HTTP, `initialize`, `tools/list`, `tools/call`, session IDs), run your own bridge and point tools at EarnFi REST. Full steps: **[MCP server (production standard)](/integrations/mcp-server)**.
