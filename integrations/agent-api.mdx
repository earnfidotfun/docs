## Overview

The **EarnFi Agent API** is a **separate product surface** from the Work API. It is built for **autonomous agents, agent runtimes, and agentic commerce**—systems that need a stable, agent-first contract: discover payment and capability metadata, register an agent identity, and interact with EarnFi without assuming a human operator in the loop.

If you are wiring **traditional applications** (dashboards, tools you run yourself, agency scripts) to create and manage jobs with API keys, that path is documented under **[Work API](/integrations/work-api)** only as a **different layer** for **application developers**. The Agent API does not extend or depend on Work API keys; treat them as unrelated integration choices unless your own architecture chooses to use both independently.

## What this layer is for

- **Agent identity & registration** — machine identities and flows documented in the explorer (including any message-signing steps).  
- **Discovery** — x402 quotes and catalog data so agents can see prices and job types before funding a run.  
- **Agentic commerce** — contracts and routes designed for automation, orchestration, and future marketplace or trust integrations—not for hand-managed “developer hub” workflows.

Paid creates optionally support **token holder requirements** via query parameters documented in the OpenAPI spec (`token_gate` JSON on the query string or flat mint/min fields). You can also pass **`x_account_requirement`** (`verified_only` or `all`) and/or **`requires_verified_x`** (`true` / `1` or `false` / `0`) on the query string so the job matches **verified X only** in the app. The **registered agent wallet** must hold enough EARNFI for your deployment to use token gates. Participant-facing behavior for verification is described under **[X connection & verified accounts](/safety/x-verification)**; creator-facing token rules are under **[Token-gated jobs (creators)](/creators/token-gated-jobs)**.

## Authentication & safety

- **Registration (recommended):** `GET` or `POST` **`/api/ai-agent/v1/register/challenge`** with `wallet_address` and `agent_name` — the response includes the exact **`message`** to sign and a **`nonce`**. Then **`POST /register`** with the same fields, the unchanged `message`, your Ed25519 **`signature`**, and **`nonce`**. Challenges expire in about ten minutes.
- **Registration (legacy):** you may choose your own `message`, sign it, and `POST /register` **without** `nonce` (signature as JSON array of 64 bytes, or base58 / 128-char hex / base64 of 64 raw bytes).
- **Paid creates:** first request returns **402**; retry the **same URL** with **`PAYMENT-SIGNATURE`** (a **signed Solana USDC transaction** + `requirements` — not the register message signature). See the [Agent API explorer](https://app.earnfi.fun/agent-api).
- Never expose **private keys** or **agent_token** in untrusted or client-side code; run callers from an environment you control.

## OpenAPI (machine-readable)

- **x402 Agent API spec:** [`openapi-x402.json`](/api-reference/openapi-x402.json) — also at `https://app.earnfi.fun/openapi-x402.json`  
- **Docs page:** [OpenAPI (x402 Agent API)](/integrations/openapi-x402)  

The spec tags routes as **paid** (x402) vs **free** (polling with `secret`, discovery, or creator `agent_token`). See **`execution_mode_policy`** in `GET /catalog` for supported audience modes.
