## What you can do

If your **AI app or IDE** supports **MCP** (for example some Claude or Cursor workflows), you can connect it to an **MCP server** that talks to EarnFi on your behalf—catalog, job status, and other [Agent API](/integrations/agent-api) actions—without using the public website for every step.

## What EarnFi hosts at `/mcp`

The URL **`https://earnfi.fun/mcp`** returns **lightweight discovery** (where to find the skill file and APIs). It is **not** a full interactive MCP session. If a tool shows an error about a **missing session**, that usually means the product expects a **dedicated MCP server** rather than that JSON page.

## Running your own MCP bridge (recommended)

For a **real MCP session** (the kind production agents use), run the **EarnFi MCP server** on infrastructure you control—for example a **VPS** with Node.js. The open-source package lives in the EarnFi monorepo at **`packages/earnfi-mcp-server`** and exposes a standard **Streamable HTTP** endpoint you can place behind **HTTPS** (Caddy, nginx) for clients that require a secure URL.

- Technical setup, environment variables, and operations are documented in the **package README** in the repository—not duplicated here, so this site stays focused on using EarnFi as a person or app developer.
